## 🚀 Featured Project

### 🛡️ [linux-autoheal-monitor](https://github.com/kapilskumbhare-cyber/linux-autoheal-monitor)

> **Modular Linux Monitoring System with Alerting, Decision Engine & Automated Self-Healing using Bash**

This is my **capstone Linux project** — not a script, but a full system built with production thinking.

Most beginners write a script that prints CPU usage. I built a system that **detects the problem, decides what to do, alerts the team, and fixes itself** — automatically.

<details>
<summary><b>📐 Click to expand System Architecture</b></summary>

```
linux-autoheal-monitor/
│
├── modules/          → Individual monitors
│   ├── cpu.sh        → CPU usage tracking
│   ├── memory.sh     → RAM usage tracking
│   ├── disk.sh       → Disk space monitoring
│   ├── service.sh    → Critical service checks
│   └── security.sh   → Security event monitoring
│
├── engine/           → Decision logic
│   └── decision.sh   → Analyzes data, detects anomalies
│
├── alert/            → Notification system
│   └── alert.sh      → Sends email alerts on threshold breach
│
├── healing/          → Auto-recovery
│   └── heal.sh       → Restarts failed services automatically
│
├── logs/             → Persistent audit logs
├── config.conf       → Configurable thresholds
└── monitor.sh        → Entry point → chmod +x && ./monitor.sh
```

</details>

| Feature | Details |
|---|---|
| 📊 Real-time Monitoring | CPU · RAM · Disk · Services · Security events |
| 🧠 Decision Engine | Analyzes system state, detects threshold breaches |
| 🚨 Alerting | Automated email notifications via Mail Utility |
| 🔁 Self-Healing | Auto-restarts failed services — zero manual intervention |
| 📝 Logging | Full audit trail for post-incident analysis |

**Stack:** `Linux` `Bash` `Cron Jobs` `Systemctl` `Mail Utility` `Git`
